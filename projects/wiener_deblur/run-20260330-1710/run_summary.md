## Lessons Learned

1. **KAPPA matters enormously**: The default KAPPA=0.01 was catastrophically wrong for a 15-pixel motion blur. For strong blurs, KAPPA needs to be orders of magnitude larger to prevent noise amplification.

2. **Blind PSNR is a degenerate metric**: The blind kernel search always found near-identity kernels (len=2-3) regardless of KAPPA or LAMBDA changes, because the loss function `MSE(re-blur) - λ·sharpness` is biased toward minimal kernels. Oracle PSNR (known true kernel) is the right measure for algorithm quality.

3. **Wiener ringing responds to blending**: A Wiener-blurred blend post-processing (`α·Wiener + (1-α)·blurred`) effectively reduces ringing artifacts. Each round of alpha reduction yielded ~0.6-0.9 dB improvement until alpha=0.74.

4. **LAMBDA is ineffective at low KAPPA**: With KAPPA=0.01, the loss landscape was completely flat — changing LAMBDA across 3 orders of magnitude had zero effect on kernel selection or output quality.
